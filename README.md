# 🍱 弁当屋経営シミュレーション (Bento Tycoon)

自由に食材を組み合わせてオリジナルのお弁当を開発し、日替わりで発生する様々なランダムイベント（猛暑、物価高騰、トレンドブームなど）に対応しながら目標金額を目指す、Webブラウザで遊べる経営シミュレーションゲームです。

## 🎮 ゲーム概要

プレイヤーは弁当屋のオーナーとなり、初期資金5,000円からスタートします。
売上を伸ばして**目標金額30,000円**を達成すればゲームクリア、資金が0円以下になると倒産（ゲームオーバー）となります。

### コア機能
1. **メニュー自由開発**: 「ごはん」「ジューシー唐揚げ」「極上焼き鮭」などの具材を自由に組み合わせ、名前をつけて新メニューを登録できます（販売価格は原価を元に自動計算されます）。
2. **リアルタイム販売**: 開発したお弁当をワンクリックで調理・販売。資金の増減がリアルタイムに反映されます。
3. **ランダムイベントシステム**: お弁当が4個売れて次の日に移行するタイミングで、天候や社会情勢などのイベントがランダムで発生。食材の仕入れ原価や弁当の売価がリアルタイムに変動します。

---

## 🚀 動作方法

特別なビルド環境（Node.js, Vite, Webpackなど）やサーバーの起動は**一切不要**です。

1. 本リポジトリの `index.html` をローカル環境にダウンロード（またはクローン）します。
2. `index.html` ファイルをブラウザ（Google Chrome, Microsoft Edge, Safariなど）にドラッグ＆ドロップするか、ダブルクリックで開きます。
3. すぐにゲームが開始されます。

## 🛠️ 技術スタックと設計のポイント

### 使用技術
* **HTML5 / CSS3** (レスポンシブ・グリッドレイアウト)
* **JavaScript (ES6+)** ※ソースコードのロジックはTypeScriptをベースにオブジェクト指向で設計

### 💡 TypeScript（設計面）でのこだわりポイント
* **厳格な型管理 (`Type / Interface`)**:
  `Ingredient`（食材）、`BentoMenu`（メニュー）、`GameEvent`（イベント）を厳密に型定義。存在しない食材の追加や、設定漏れによるランダムイベントのバグをコンパイル段階で防ぐ設計にしています。
* **動的な状態（時価）計算機能**:
  お弁当のデータ構造には固定の「基本価格」のみを持たせ、画面描画や販売処理の瞬間に「現在のイベント倍率」を掛け算する設計にしています。これにより、マスターデータを汚さずに柔軟な時価システムを実現しました。
* **インデックス型（ディクショナリ型）の活用**:
  イベントシステムにおける食材ごとのコスト変動には `{ [key: string]: number }` を採用。「特定の食材ID（例: salmon）のコストだけを2.5倍にする」といった拡張性の高いデータ構造にしています。
